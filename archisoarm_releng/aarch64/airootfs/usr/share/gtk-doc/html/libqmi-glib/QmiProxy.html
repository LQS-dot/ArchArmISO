Description

The QmiProxy will setup an abstract socket listening on a predefined address, and will take care of synchronizing the access to a set of shared QMI ports.

Multiple QmiDevice objects may be connected to the QmiProxy at any given time. The QmiProxy acts as a stateless proxy for non-CTL services (messages are transferred unmodified), and as a stateful proxy for the CTL service (all remote QmiDevice objects will need to share the same CTL message sequence ID).
